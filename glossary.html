01 שכבה ראשונה

🏠 איפה אתם עובדים

החדר, השולחן, והבית של Claude.

🖥️
VS Codeהחדר שבו עובדים
בעברית פשוטה

התוכנה שפותחים על המחשב כדי לעבוד. החדר שבו הכול קורה.

כמו ש...

Word הוא החדר שבו כותבים מסמך — VS Code הוא החדר שבו עובדים עם Claude ועם הקבצים שלכם.

מתי תפגשו אותו

כל פעם שתתחילו לעבוד. פותחים את VS Code — וממנו הכול מתחיל.

⌨️
Terminal (טרמינל)הבית של Claude Code
בעברית פשוטה

החלון שבו Claude Code חי ועובד — המקום שבו הוא מריץ את כל הפקודות שאתם מבקשים ממנו.

כמו ש...

המטבח של מסעדה. אתם מזמינים את המנה (מבקשים מ-Claude), והשף נכנס למטבח ומבשל — בלי שתצטרכו להיכנס למטבח בעצמכם.

מתי תפגשו אותו

תראו אותו עובד — טקסט רץ בחלון בזמן ש-Claude מבצע את הבקשה. זה תקין, זה פשוט הוא עובד. אתם לא צריכים לכתוב שם כלום בעצמכם.

😌 אל תדאגו — זה לא בשבילכם לכתוב קוד

אתם לא מתכנתים, ולא צריכים לכתוב שורות קוד בטרמינל כדי שדברים ירוצו. אתם פשוט כותבים ל-Claude Code, במילים שלכם, מה אתם רוצים — והוא מריץ את הפקודות בטרמינל בשבילכם. ואם הוא מבקש מכם להריץ משהו? פשוט אמרו לו: "תריץ אתה" — והוא יעשה את זה. הטרמינל הוא שלו לעבוד בו, לא שלכם.

☁️
מקומי (לוקאלי) מול בענןאיפה Claude באמת רץ
בעברית פשוטה

שתי דרכים להפעיל את Claude — על המחשב שלכם ממש (מקומי), או על מחשב מרוחק של Anthropic שניגשים אליו מהדפדפן (בענן).

כמו ש...

עובד שיושב פיזית במשרד שלכם וניגש לארונות שלכם (מקומי) — לעומת עובד שיושב במשרד אחר ועובד על עותק של התיקים ששלחתם לו (בענן).

כמו ש...

מנוע ברכב — אף פעם לא רואים אותו, לא נוגעים בו, אבל בלעדיו האוטו לא זז.

מתי תפגשו אותו

כמעט אף פעם. הותקן פעם אחת בהתקנה, ומאז פשוט עובד ברקע. טוב לדעת שהוא שם.

02 שכבה שנייה

🤝 מי עוזר לכם

העובד שמולכם, והמוח שבחרתם לו.

🤖
Claude Codeהעובד שמבצע בשבילכם
בעברית פשוטה

העוזר עצמו. לא סתם בוט שמייעץ — עובד שמבצע. כותב, מתקן, פותח קבצים, בונה דברים בשבילכם.

כמו ש...

עובד חדש שיושב מולכם, שאתם מסבירים לו מה צריך — והוא עושה. ההבדל: הוא לא הולך הביתה ולא שוכח.

מתי תפגשו אותו

זה הלב של הכול. כל מה שתבקשו — תבקשו ממנו בעברית רגילה, והוא יבצע. (איפה הוא מבצע את העבודה? בטרמינל — חדר העבודה שלו.)

🧠
המודל (Opus / Sonnet / Haiku)המוח של העוזר
בעברית פשוטה

ה"מוח" שבחרתם לעוזר. יש כמה רמות: חכם ויסודי (Opus), מהיר ומאוזן (Sonnet), זריז וקליל (Haiku).

כמו ש...

אותו עובד, אבל אתם בוחרים אם הוא עובד במצב "ריכוז מלא" או "מהיר וקליל". משימה מורכבת — מוח חזק. משימה פשוטה — מוח מהיר.

מתי תפגשו אותו

כשתכתבו /model ותבחרו. ההמלצה לסדנה: Opus — החזק ביותר.

03 שכבה שלישית

📁 מה יש לכם במחשב

מהדבר הקטן ביותר ועד עבודה שלמה.

📄
קובץ (File)דף בודד
בעברית פשוטה

פריט בודד — מסמך, תמונה, שורת קוד. הדבר הכי קטן שעובדים עליו.

כמו ש...

דף בודד בקלסר.

מתי תפגשו אותו

כל הזמן. כשתבקשו מ-Claude "תכתוב לי..." או "תתקן את...", הוא עובד על קובץ.

📝
קובץ .md (Markdown)מסמך טקסט קליל
בעברית פשוטה

סוג קובץ טקסט פשוט שאפשר להוסיף בו עיצוב קל — כותרות, רשימות, הדגשות — בלי תוכנה מסובכת.

כמו ש...

דף Word, אבל קליל וגמיש, שכל מחשב פותח בקלות. (גם המדריך הזה הוא קובץ .md.)

מתי תפגשו אותו

רוב המסמכים שתעבדו עליהם עם Claude הם .md — זה הפורמט שהוא הכי אוהב לכתוב ולקרוא בו.

📋
CLAUDE.mdתיק החפיפה של Claude
בעברית פשוטה

קובץ .md מיוחד ש-Claude קורא אוטומטית בכל פעם שהוא מתחיל לעבוד בפרויקט. שם רושמים לו, פעם אחת, את כל מה שחשוב שידע — על העסק, על הסגנון, על הכללים.

כמו ש...

תיק חפיפה לעובד חדש. במקום להסביר כל בוקר מי הלקוחות ואיך עובדים כאן — כותבים פעם אחת, והעובד קורא לבד וכבר יודע.

מתי תפגשו אותו

בהמשך נבנה לכם CLAUDE.md משלכם — קובץ שמתאר את העסק שלכם, הלקוחות וטון הדיבור. מאותו רגע, Claude "מכיר" את העסק בלי שתצטרכו להסביר כל פעם מחדש. זה ההבדל בין עוזר זר לבין עובד שכבר עבר חפיפה.

🔑
.env (קובץ הסודות)כספת המפתחות
בעברית פשוטה

קובץ קטן וסודי שבו שומרים את כל ה"מפתחות" — הטוקנים והסיסמאות שמחברים את Claude לשירותים חיצוניים (וואטסאפ, חשבוניות, Make). במקום לכתוב את הסוד בתוך הקוד, שמים אותו ב-.env, ו-Claude קורא אותו משם.

כמו ש...

צרור המפתחות של העסק. מי שמחזיק בו נכנס לכל הדלתות — אז לא משאירים אותו על השולחן, לא מצלמים, ולא שולחים בוואטסאפ.

מתי תפגשו אותו

כשתחברו את Claude לשירות חיצוני (בוט, חשבוניות, אוטומציה). הטוקן נכנס ל-.env, ו-Claude קורא אותו משם — בלי שהסוד מופיע בקוד עצמו.

⚠️ איך עובדים איתו נכון — קריטי

הקובץ הכי רגיש שיש. שלושה כללי ברזל: (1) אף פעם לא מעלים אותו ל-GitHub — יש קובץ מיוחד בשם .gitignore שאומר ל-Git "את .env תתעלם, אל תשלח לענן לעולם". (2) לא משתפים, לא שולחים, לא מצלמים את התוכן. (3) אם טוקן נחשף בטעות — מבטלים אותו מיד ומוציאים חדש. דליפה של .env = מישהו זר נכנס לכל המערכות שלכם בבת אחת.

04 שכבה רביעית

🔐 איך שומרים הכל

השכבה שהכי מבלבלת — נלך לאט.

🎞️
Git (גיט)מצלמת אבטחה לעבודה
בעברית פשוטה

מצלמת אבטחה לעבודה שלכם. שומרת תמונה של כל שלב — כדי שתמיד אפשר לחזור אחורה.

כמו ש...

כפתור "בטל" (Undo) בלי הגבלה. לא רק הפעולה האחרונה — אלא כל גרסה שהייתה אי-פעם, מסודרת לפי זמן.

מתי תפגשו אותו

הוא עובד ברקע. לא חייבים לגעת בו — אבל בזכותו אף שינוי לא הולך לאיבוד, וגם אם משהו מתקלקל, אפשר לחזור לרגע שבו הכול עבד.

📸
Commit (קומיט)נקודת שמירה
בעברית פשוטה

"צילום מצב" — נקודה שאתם שומרים ואומרים "לכאן אני יכול לחזור".

כמו ש...

לשמור משחק במחשב לפני שלב קשה. אם נכשלתם — חוזרים לנקודת השמירה במקום להתחיל מהתחלה.

מתי תפגשו אותו

בכל פעם שסיימתם חתיכת עבודה משמעותית. Claude יכול לעשות את זה בשבילכם — מספיק לבקש "תשמור נקודה".

☁️
GitHub (גיטהאב)הכספת בענן
בעברית פשוטה

אתר באינטרנט ששומר עותק של הפרויקטים שלכם — גיבוי בענן, מחוץ למחשב.

כמו ש...

Google Drive או Dropbox, אבל לקוד ולפרויקטים. גם אם המחשב יישבר או ייגנב — העבודה שמורה שם.

מתי תפגשו אותו

כשתרצו לגבות פרויקט, לעבוד עליו מכמה מחשבים, או לשתף אותו. (לדוגמה: סקריפט ההתקנה של הסדנה יושב ב-GitHub — ומשם התלמידים מורידים אותו.)

🗄️
Repo (ריפו)הקלסר של פרויקט
בעברית פשוטה

הקלסר של פרויקט אחד בתוך GitHub — עם כל הגרסאות הישנות שמורות בפנים.

כמו ש...

קלסר שלא שומר רק את הדף הנוכחי, אלא כל טיוטה שהייתה אי-פעם, מסודרת לפי תאריך.

מתי תפגשו אותו

כל פרויקט שתעלו ל-GitHub יושב ב-repo משלו. כשמישהו אומר "תכניס את זה ל-repo" — הכוונה: "תשים את הפרויקט הזה בקלסר שלו בענן".

מילה אחת על הבלבול הכי נפוץ:  Git ≠ GitHub.
Git = מצלמת האבטחה במחשב שלכם (מקומי). GitHub = הכספת באינטרנט (בענן). Git עובד גם בלי אינטרנט; ל-GitHub מעלים כשרוצים גיבוי בענן.
⬆️
Push — מתי ולמה מעליםלהעלות מהמחשב לענן
תוך כדי עבודה

Git (המצלמה במחשב) שומר הכול אוטומטית. לא צריך אינטרנט, לא צריך כלום.

בסוף יום / סיום שלב / רוצים גיבוי

מעלים ל-GitHub. הפעולה נקראת Push ("דחיפה" — דוחפים את העותק מהמחשב אל הענן).

חוק האצבע

אם המחשב שלכם ייעלם עכשיו — מה שלא העליתם ל-GitHub קיים רק בו. סיימתם משהו שחשוב לכם? תעלו לענן. ואת זה לא צריך לעשות לבד — פשוט בקשו מ-Claude: "תעלה את זה ל-GitHub".
