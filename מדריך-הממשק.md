# 📖 מילון הממשק — כל המילים המוזרות, בעברית פשוטה

> **למי זה מיועד?** לך — בעל/ת עסק, לא מתכנת/ת.
> **מה זה עושה?** לא מלמד פקודות. רק מסביר, במילים של בן אדם, מה כל דבר שאתה רואה ושומע על המסך — כדי שתפסיק/י לפחד מהמסך השחור.
>
> אין כאן מה לזכור בעל פה. כשתיתקל/י במילה שלא מובנת — תחזור/י לכאן, תקרא/י שתי שורות, ותמשיך/י.

---

## 🗺️ מפת המסע — איך הכול מתחבר

לפני המילים, התמונה הגדולה. כשעובדים עם Claude Code, יש 4 שכבות:

| שכבה                  | מה זה                  | בקצרה                    |
| --------------------- | ---------------------- | ------------------------ |
| **1. איפה עובדים**    | VS Code · Terminal     | החדר והשולחן שלך         |
| **2. מי עוזר לי**     | Claude Code · המודל    | העובד שמולך, והמוח שלו   |
| **3. מה יש במחשב**    | קובץ · תיקייה · פרויקט | הניירת שעובדים עליה      |
| **4. איך שומרים הכל** | Git · GitHub · Repo    | מצלמת האבטחה והכספת בענן |

עכשיו נעבור עליהן אחת-אחת. כל מושג בנוי אותו דבר: **מה זה** → **כמו ש...** → **מתי תפגוש אותו**.

## שכבה 1 — איפה עובדים 🏠

### 🖥️ VS Code

**בעברית פשוטה:** התוכנה שפותחים על המחשב כדי לעבוד. החדר שבו הכול קורה.

**כמו ש...** Word זה החדר שבו כותבים מסמך — VS Code זה החדר שבו עובדים עם Claude ועם הקבצים שלך.

**מתי תפגוש אותו:** כל פעם שתתחיל/י לעבוד. פותחים את VS Code — וממנו הכול מתחיל.

---

### ⌨️ Terminal (טרמינל)

**בעברית פשוטה:** החלון שבו Claude Code חי ועובד — המקום שבו **הוא** מריץ את כל הפקודות שאת/ה מבקש/ת ממנו.

**כמו ש...** המטבח של מסעדה. את/ה מזמין/ה את המנה (מבקש/ת מ-Claude), והשף נכנס למטבח ומבשל — בלי שתצטרך/י להיכנס למטבח בעצמך.

**מתי תפגוש אותו:** תראה/י אותו עובד — טקסט רץ בחלון בזמן ש-Claude מבצע את הבקשה. זה תקין, זה פשוט הוא עובד. את/ה לא צריך/ה לכתוב שם כלום בעצמך.

**אל תדאג/י — זה לא בשבילך לכתוב קוד:** את/ה _לא_ מתכנת/ת, ו*לא* צריך/ה לכתוב שורות קוד בטרמינל כדי שדברים ירוצו. את/ה פשוט כותב/ת ל-Claude Code, במילים שלך, מה את/ה רוצה — **והוא** מריץ את הפקודות בטרמינל בשבילך. ואם הוא מבקש ממך להריץ משהו בטרמינל? פשוט תגיד/י לו: **"תריץ אתה"** — והוא יעשה את זה. הטרמינל הוא שלו לעבוד בו, לא שלך.

---

### 🖥️ מקומי (לוקאלי) מול ☁️ בענן

**בעברית פשוטה:** שתי דרכים להפעיל את Claude — על המחשב שלך ממש (**מקומי**), או על מחשב מרוחק של Anthropic שניגשים אליו מהדפדפן (**בענן**).

**כמו ש...** עובד שיושב **פיזית במשרד שלך** וניגש לארונות שלך (מקומי) — לעומת עובד שיושב **במשרד אחר** ועובד על עותק של התיקים ששלחת לו (בענן).

**ההבדל בפועל:**

|                        | 🖥️ מקומי               | ☁️ בענן                        |
| ---------------------- | ---------------------- | ------------------------------ |
| על אילו קבצים עובדים   | האמיתיים, על המחשב שלך | עותק מ-GitHub                  |
| המחשב צריך להיות דלוק? | כן                     | לא                             |
| מאיפה ניגשים           | VS Code על המחשב       | דפדפן / טלפון, מכל מקום        |
| הכי מתאים ל...         | עבודה רגילה, הסדנה     | עבודה מהדרך, כמה משימות במקביל |

**מתי תפגוש אותו:** בסדנה את/ה עובד/ת **מקומי** — הכי פשוט, הקבצים מולך. הענן זה שדרוג להמשך, כשרוצים לעבוד מהטלפון או בלי שהמחשב תפוס. (וזו עוד סיבה ש-GitHub חשוב — הענן עובד מול ה-repo שם.)

---

### ⚙️ Node

**בעברית פשוטה:** מנוע ש"רץ" מתחת לפני השטח ומאפשר ל-Claude Code לעבוד.

**כמו ש...** מנוע ברכב — אף פעם לא רואים אותו, לא נוגעים בו, אבל בלעדיו האוטו לא זז.

**מתי תפגוש אותו:** כמעט אף פעם. הוא הותקן פעם אחת בהתקנה, ומאז הוא פשוט עובד ברקע. טוב לדעת שהוא שם.

## שכבה 2 — מי עוזר לי 🤝

### 🤖 Claude Code

**בעברית פשוטה:** העוזר עצמו. לא סתם בוט שמייעץ — עובד שמבצע. כותב, מתקן, פותח קבצים, בונה דברים בשבילך.

**כמו ש...** עובד חדש שיושב מולך, שאת/ה מסביר/ה לו מה צריך — והוא עושה. ההבדל: הוא לא הולך הביתה ולא שוכח.

**מתי תפגוש אותו:** זה הלב של הכול. כל מה שתבקש/י — תבקש/י ממנו בעברית רגילה, והוא יבצע. (איפה הוא מבצע את העבודה? בטרמינל — חדר העבודה שלו.)

---

### 🧠 המודל (Opus / Sonnet / Haiku)

**בעברית פשוטה:** ה"מוח" שבחרת לעוזר. יש כמה רמות: חכם ויסודי (Opus), מהיר ומאוזן (Sonnet), זריז וקליל (Haiku).

**כמו ש...** אותו עובד, אבל את/ה בוחר/ת אם הוא עובד במצב "ריכוז מלא" או "מהיר וקליל". למשימות מורכבות — מוח חזק. למשימות פשוטות — מוח מהיר.

**מתי תפגוש אותו:** כשתכתוב/י `/model` ותבחר/י. ההמלצה לסדנה: **Opus** — החזק ביותר.

---

## שכבה 3 — מה יש במחשב 📁

### 📄 קובץ (File)

**בעברית פשוטה:** פריט בודד — מסמך, תמונה, שורת קוד. הדבר הכי קטן שעובדים עליו.

**כמו ש...** דף בודד בקלסר.

**מתי תפגוש אותו:** כל הזמן. כשתבקש/י מ-Claude "תכתוב לי..." או "תתקן את...", הוא עובד על קובץ.

---

### 📝 קובץ .md (Markdown)

**בעברית פשוטה:** סוג קובץ טקסט פשוט שאפשר להוסיף בו עיצוב קל — כותרות, רשימות, הדגשות — בלי תוכנה מסובכת.

**כמו ש...** דף Word, אבל קליל וגמיש, שכל מחשב פותח בקלות. (גם המדריך הזה שאת/ה קורא/ת עכשיו הוא קובץ .md.)

**מתי תפגוש אותו:** רוב המסמכים שתעבוד/י עליהם עם Claude הם .md — זה הפורמט שהוא הכי אוהב לכתוב ולקרוא בו.

---

### 📋 CLAUDE.md

**בעברית פשוטה:** קובץ .md **מיוחד** ש-Claude קורא **אוטומטית** בכל פעם שהוא מתחיל לעבוד בפרויקט. שם רושמים לו, פעם אחת, את כל מה שחשוב שידע — על העסק, על הסגנון, על הכללים.

**כמו ש...** תיק חפיפה לעובד חדש. במקום להסביר כל בוקר מי הלקוחות ואיך עובדים כאן — כותבים פעם אחת, והעובד קורא לבד וכבר יודע.

**מתי תפגוש אותו:** בהמשך נבנה לך CLAUDE.md משלך — קובץ שמתאר את העסק שלך, הלקוחות וטון הדיבור. מאותו רגע, Claude "מכיר" את העסק בלי שתצטרך/י להסביר כל פעם מחדש. זה ההבדל בין עוזר זר לבין עובד שכבר עבר חפיפה.

---

### 🔑 .env (קובץ הסודות)

**בעברית פשוטה:** קובץ קטן וסודי שבו שומרים את כל ה"מפתחות" — הטוקנים והסיסמאות שמחברים את Claude לשירותים חיצוניים (וואטסאפ, חשבוניות, Make וכו'). במקום לכתוב את הסוד בתוך הקוד, שמים אותו ב-`.env`, ו-Claude קורא אותו משם.

**כמו ש...** צרור המפתחות של העסק. מי שמחזיק בו נכנס לכל הדלתות — אז לא משאירים אותו על השולחן, לא מצלמים אותו, ולא שולחים בוואטסאפ.

**מתי תפגוש אותו:** כשתחבר/י את Claude לשירות חיצוני (בוט, חשבוניות, אוטומציה). הטוקן נכנס ל-`.env`, ו-Claude קורא אותו משם — בלי שהסוד מופיע בקוד עצמו.

**⚠️ איך עובדים איתו נכון — קריטי:** הקובץ הכי רגיש שיש. שלושה כללי ברזל: **(1)** אף פעם לא מעלים אותו ל-GitHub — יש קובץ מיוחד בשם `.gitignore` שאומר ל-Git "את `.env` תתעלם, אל תשלח לענן לעולם". **(2)** לא משתפים, לא שולחים, לא מצלמים את התוכן. **(3)** אם טוקן נחשף בטעות — מבטלים אותו מיד ומוציאים חדש. דליפה של `.env` = מישהו זר נכנס לכל המערכות שלך בבת אחת.

---

### 🗂️ תיקייה (Folder)

**בעברית פשוטה:** קופסה שמכילה כמה קבצים יחד.

**כמו ש...** מגירה בארון שבה שמים כמה דפים שקשורים זה לזה.

**מתי תפגוש אותה:** כשהעבודה שלך מתחילה לגדול — מסדרים את הקבצים בתיקיות לפי נושא.

---

### 📦 פרויקט (Project)

**בעברית פשוטה:** התיקייה הראשית של עבודה אחת שלמה — עם כל הקבצים והתיקיות שלה.

**כמו ש...** התיק המלא של לקוח אחד: כל המסמכים, החוזים והתכתובות שלו ביחד במקום אחד.

**מתי תפגוש אותו:** כשתפתח/י משהו חדש לעבוד עליו — הסדנה, דף נחיתה, בוט — כל אחד הוא פרויקט נפרד.

---

## שכבה 4 — איך שומרים הכל 🔐

זו השכבה שהכי מבלבלת אנשים. נלך לאט.

### 🎞️ Git (גיט)

**בעברית פשוטה:** מצלמת אבטחה לעבודה שלך. שומרת תמונה של כל שלב — כדי שתמיד אפשר לחזור אחורה.

**כמו ש...** כפתור "בטל" (Undo) בלי הגבלה. לא רק את הפעולה האחרונה — אלא כל גרסה שהייתה אי-פעם, מסודרת לפי זמן.

**מתי תפגוש אותו:** הוא עובד ברקע. את/ה לא חייב/ת לגעת בו — אבל בזכותו אף שינוי לא הולך לאיבוד, וגם אם משהו מתקלקל, אפשר לחזור לרגע שבו הכול עבד.

---

### 📸 Commit (קומיט)

**בעברית פשוטה:** "צילום מצב" — נקודה שאת/ה שומר/ת ואומר/ת "לכאן אני יכול/ה לחזור".

**כמו ש...** לשמור משחק במחשב לפני שלב קשה. אם נכשלת — חוזרים לנקודת השמירה במקום להתחיל מהתחלה.

**מתי תפגוש אותו:** בכל פעם שסיימת חתיכת עבודה משמעותית. Claude יכול לעשות את זה בשבילך — מספיק לבקש "תשמור נקודה".

---

### ☁️ GitHub (גיטהאב)

**בעברית פשוטה:** אתר באינטרנט ששומר עותק של הפרויקטים שלך — גיבוי בענן, מחוץ למחשב.

**כמו ש...** Google Drive או Dropbox, אבל לקוד ולפרויקטים. גם אם המחשב יישבר או ייגנב — העבודה שמורה שם.

**מתי תפגוש אותו:** כשתרצה/י לגבות פרויקט, או לעבוד עליו מכמה מחשבים, או לשתף אותו. (לדוגמה: סקריפט ההתקנה של הסדנה יושב ב-GitHub שלך — ומשם התלמידים מורידים אותו.)

---

### 🗄️ Repo (ריפו)

**בעברית פשוטה:** הקלסר של פרויקט אחד בתוך GitHub — עם כל הגרסאות הישנות שמורות בפנים.

**כמו ש...** קלסר שלא שומר רק את הדף הנוכחי, אלא כל טיוטה שהייתה אי-פעם, מסודרת לפי תאריך.

**מתי תפגוש אותו:** כל פרויקט שתעלה/י ל-GitHub יושב ב-repo משלו. כשמישהו אומר "תכניס את זה ל-repo" — הכוונה: "תשים את הפרויקט הזה בקלסר שלו בענן".

> **מילה אחת על הבלבול:** `Git` ≠ `GitHub`.
> **Git** = מצלמת האבטחה במחשב שלך (מקומי). **GitHub** = הכספת באינטרנט (בענן). Git עובד גם בלי אינטרנט; GitHub זה לשם מעלים כשרוצים גיבוי בענן.

---

## ⬆️ מתי ולמה מעלים ל-GitHub?

זאת השאלה הכי נפוצה. בפשטות:

- **תוך כדי עבודה** → Git (המצלמה במחשב) שומר הכול אוטומטית. לא צריך אינטרנט, לא צריך כלום.
- **בסוף יום / סיום שלב חשוב / רוצה גיבוי בענן** → מעלים ל-GitHub. הפעולה הזאת נקראת **Push** ("דחיפה" — דוחפים את העותק מהמחשב אל הענן).

**חוק האצבע:** אם המחשב שלך ייעלם עכשיו — מה שלא העלית ל-GitHub, קיים רק שם. לכן: סיימת משהו שחשוב לך? תעלה/י לענן.

ואת זה גם לא צריך לעשות לבד — אפשר פשוט לבקש מ-Claude: _"תעלה את זה ל-GitHub"_.
